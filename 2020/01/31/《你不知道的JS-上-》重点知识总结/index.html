《你不知道的JS(上)》重点知识总结

Beking's notebook
2020-01-31
| 阅读量 296
JavaScript

阅读摘记《你不知道的JS(上)》 | 仅供学习

  • 编译器将程序分解成词法单元 -> 将词法单元解析成一个🌲结构
  1. 询问作用域：是否已存在该名称的变量？是，则忽略该声明；否，则声明一个新变量并命名。
  2. 询问作用域：是否已存在该名称的变量？是，则使用该变量；否，则继续查找
  • 当变量出现在赋值操作的左侧时进行 LHS 查询(试图找到变量的容器本身)，否则进行 RHS 查询(查找某个变量的值)。
  • 不成功的 LHS 引用：自动隐式创建一个全局变量，严格模式下 ReferenceError。
  • 不成功的 RHS 引用：ReferenceError。
  • 严格模式下禁止自动或隐式创建全局变量。
  • 作用域查找会在 找到第一个匹配的标识符时 停止(由内而外)。
  • 遮蔽效应：在多层的嵌套作用域中可以定义同名的标识符，内部遮蔽外部。
  • 函数的词法作用域只由这 函数被声明时所处位置 决定。
  • 欺骗词法作用域(使用 eval 或 with)会导致性能下降。
  • eval() 用于执行动态创建的代码，可以在运行期修改书写期的词法作用域。
  • eval() 若 接受了一个或多个声明代码 ，则会修改其所处的词法作用域。
  • 严格模式下，eval() 在运行时有自己的词法作用域，其中的声明无法修改所在作用域。类似的还有：setTimeout(), setInterval(), new Function()等。
  • with 用于重复引用同一个对象中的多个属性，可以将一个没有或有多个属性的对象处理为一个 完全隔离 的词法作用域。
  • with 声明会根据传递的对象 凭空创建一个全新的词法作用域 。
  • 函数声明和函数表达式的区别： 名称标识符绑定处 不同。
  • 函数声明不可忽略函数名，函数表达式可以匿名。
  • 匿名函数表达式缺点：调试困难；只能通过已弃用的 arguments.callee 引用自身；可读性差。
  • 临时性死区：let 声明不会在块作用域中提升，声明的代码被运行前声明并不存在。可以避免全局污染，有利于垃圾回收。
  • 提升：变量和函数声明从它们在代码中出现的位置提升到顶部。所有声明会在任何代码被执行前首先被处理。
  • 函数优先提升，其次是变量。重复声明忽略，重复函数覆盖前面的。
  • for(let i = 0;;) for 循环头部的 let 不仅将 i 绑定到了 for 循环的块中，事实上它将 i 重新绑定到了循环的每一个迭代中。
  • 闭包优点：
  1. 可以让变量保存在内存中，不被垃圾回收机制清除。
  2. 可以避免变量的全局污染 。
  3. 可以定义模块，将操作函数暴露到外部，细节隐藏在内部。
  • 闭包缺点：
  1. 易造成内存泄漏。
  2. 闭包会降低性能。
  • 当函数可以记住并访问所在的词法作用域 时，就产生了闭包。
  • 只要 使用了回调函数 就是在使用闭包，例如：定时器，事件监听器，ajax请求，IIFE，跨窗口通信等。
  • 延迟函数的回调会在循环结束时执行。
  • 循环中的函数是 在各个迭代中分别定义的 ，但都被封闭在一个共享的全局作用域中，即所有函数共享一个 i 的引用。
  • IIFE 会使每个迭代都生成一个新的作用域，使延迟函数的回调可以将新的作用域封闭在每个迭代内部。
  • 动态作用域不关心函数和作用域如何声明、何处声明，只关心从何处调用。
  • 词法作用域：声明处；动态作用域：调用处。
  • this 在任何情况下都不指向函数的词法作用域。
  • this 是在函数被调用时发生绑定，指向完全取决于调用处。
  • this 的绑定规则：
  1. 默认绑定：非严格模式下默认绑定在全局对象，严格模式下 undefined。
  2. 隐式绑定：调用位置是否有上下文，被隐式绑定的函数会丢失绑定的对象，就上一层调用位置起作用。
  3. 显式绑定：硬绑定(call，bind，apply，forEach)，硬绑定后不可能再改变它的 this。软绑定(softBind)，保留隐式绑定或是显式绑定修改 this 的能力。
  4. new 绑定：调用者绑定。
  5. 箭头函数：继承父作用域的 this，相当于 self=this 的作用。
  • new 的过程：构建一个全新的对象 -> 新对象执行 [[prototype]] 连接 -> 新对象绑定到函数调用的 this -> 返回此新对象
  • 装箱：传入一个原始值作为 this 的绑定对象时，原始值 -> 包装类型。
  • bind：创建一个新的包装函数，忽略当前的 this 绑定，把对象绑定到 this 上。
  • 内置对象：String, Number, Boolean, Object, Function, Array, Date, RegExp, Error。
  • 判断类型：typeof, instanceOf, Object.prototype.toString.call()
  • null 和 undefined 和 Date 无构造形式，仅文字形式。
  • 属性名必是字符串，可计算属性名可应用于 Symbol。
  • Object.defineProperty(obj, 'name', {}) 添加一个新属性或修改一个已有属性。
  • 常量：writable:false; configurable:false;
  • Object.preventExtensions() 禁止向一个对象添加新属性。
  • Object.seal() 密封，不可增删，不可重新配置，可以修改。
  • Object.freeze() 冻结，不可增删改，不可重新配置，是最高的不可变性。
  • 区分属性是否可枚举：
  1. propertyIsEnumerable 只检查对象，不检查原型链，enumerable:true。
  2. Object.keys 返回数组，包含所有可枚举属性，只对象。
  3. Object.getOwnPropertyNames 返回数组，包含所有属性，无论是否可枚举，只对象。
  4. in 检查对象和原型链。
  5. hasOwnProperty 只检查对象。
  • in 和 hasOwnProperty 的区别： 是否检查原型链 。
  • 迭代器：
  1. for-in 遍历对象的可枚举属性列表(包括原型链)。
  2. forEach 遍历数组中所有值并忽略回调函数的返回值。
  3. for-of 遍历有 iterator 的对象。
  • 面向对象：封装、继承、多态 。
  • 多态(在继承链的不同层次名称相同但是功能不同的函数)，在继承链的不同层次中的一个方法可以被多次定义。
  • super 可以直接调用父类的构造函数。
  • 类的继承其实是复制。
  • 所有普通对象都有内置的 Object.prototype，指向原型链的顶端。所有普通的原型链最终都会指向内置的 Object.prototype 。
  • 屏蔽属性：只读属性会阻止原型链下层隐式创建同名属性(为了模拟类属性的继承)。
  • new Foo() 并没有直接创建关联，只是间接完成了一个关联到其他对象的新对象。
  • 继承：意味着复制。JS 默认并不会复制对象属性，而是会在两个对象之间创建一个关联，这样一个对象就可以通过委托访问另一个对象的属性和函数 。
  • new 会劫持所有普通函数并用构造对象的形式来调用它，成为构造函数调用 。
  • Object.create(a) 以 a 为原型创建一个对象、创建一个新的对象并将它关联到 a。
  • Object.create 的唯一缺点：不能直接修改已有的默认对象。
  • Object.setPropertyOf() 直接修改现有的某个原型。
  • a instanceOf b 在 a 的原型链上是否有 a.prototype 指向的对象。
  • b.isPrototypeOf(c) b 是否出现在 c 的原型链上。
  • Object.getPrototypeOf(a) 直接获取一个对象的原型链。
  • Object.create(null) 会创建一个拥有空原型的对象，无法委托，没有原型链。
  • 要访问对象中不存在的一个属性时，[[Get]] 就会查找对象内部原型关联的对象。此关联关系定义了一条原型链，在查找时对它进行遍历 。
  • 委托行为：某些对象在找不到属性或方法引用时会把请求委托给另一个对象。
  • 委托行为认为对象之间是兄弟关系，互相委托。
  • 原型机制本质上是行为委托机制，对象间兄弟关系。
  • class 基本是现有的委托机制的语法糖，不是静态复制。
  • 可以通过 super 来实现相对多态，任何方法都可以引用原型链上层的同名方法 。
  • super 是在声明时静态绑定的，并非动态绑定。
  • super 并不像 this 一样是动态绑定，而是在创建时静态绑定。
