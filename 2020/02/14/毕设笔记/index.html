技术选型 scrapy vs requests+beautifulsoup

  1. requests 和 beautifulsoup 都是库，scrapy 是框架；
  2. scrapy 框架中可以加入requests 和 beautifulsoup；
  3. scrapy 基于 twisted，性能是最大优势；
  4. scrapy 方便扩展，提供了很多内置的功能；
  5. scrapy 内置的 css 和 xpath selector 非常方便，beautifulsoup 最大的缺点就是慢。

URL去重策略

  1. 将访问过的URL保存到数据库中；
  2. 将访问过的URL保存到set中，只需要O(1)的代价就可以查询URL；
  3. URL经过md5等方法哈希后保存到set中；
  4. 用bitmap方法将访问过的URL通过hash函数映射到某一位；
  5. bloomfilter方法对bitmap进行改进，多重hash函数降低冲突。

字符串编码 encode decode

  1. 计算机只能处理数字，文本转换为数字才能处理。计算机中8个bit作为一个字节，所以一个字节能表示最大的数字就是255；
  2. ASCII(一个字节)编码就成为美国人的标准编码；
  3. ASCII处理中文是不够的，中国制定了GB2312编码，用两个字节表示一个汉字；
  4. unicode的出现将所有语言统一到一套编码里；
  5. 乱码问题解决了，但是如果内容全是英文，unicode编码比ASCII需要多一倍的存储空间，同时如果传输需要多一倍的传输；
  6. 可变长的编码utf-8的出现，把英文变长一个字节，汉字三个字节。特别生僻的变成4-6字节，如果传输大量的英文，utf-8作用就很明显了。

scrapy

scrapy 是 Python 开发的一个快速高层次的屏幕抓取和web抓取框架，用于抓取web站点并从页面中提取结构化的数据。优点：高并发(底层是异步IO框架 时间循环+回调)。

官方文档

  1. 下载：pip install Scrapy
  2. 新建：scrapy startproject namexxx

xpath 语法 res.xpath(‘’).extract_first(‘’)

  1. xpath 使用路径表达式在xml和html中进行导航；
  2. xpath 包含标准函数库；
  3. xpath 是一个w3c的标准。
表达式 说明
title 选取所有title的所有子节点
/title 选取根元素title
title/a 选取所有属于title的子元素的a元素
//div 选取所有div子元素
title//div 选取所有属于title的后代的div
//@class 选取所有名为class的属性
/div/* 选取属于div的所有子节点
//* 选取所有元素
//div[@*] 选取所有带属性的title元素
/div/a | //div/p 选取所有div的a和p元素
//span | //ul 选取文档中的span和ul元素
title/div/p | //span 选取所有属于title的div的p 以及所有的span

css 选择器 res.css(‘’).extract_first(‘’)

表达式 说明
* 选择所有节点
#container 选择id为container的节点
.container 选择所有class包含container的节点
li a 选择所有li下的所有a
ul+p 选择ul后面的第一个p元素
div#container>ul 选择我id为container的div的ul子元素
p~ul 选择前面有p的每个ul
a[title] 选择所有有title属性的a
a[href=”htt://xxx.com”] 选择所有href属性为badu.com的a
a[href*=”baidu”] 选择所有href属性包含baidu的a
a[href^=”http”] 选择href属性值以http开头的a
a[href$=”.jpg”] 选择所有href属性值以.jpg结尾的a
input[type=radio]:checked 选择选中的radio的元素
div:not(#container) 选择所有id非container的div
li:nth-child(3) 选择第三个li
te:nth-child(2n) 选择第偶数个tr

分布式爬虫的优点

  1. 充分利用多机器的带宽加速爬取；
  2. 充分利用多机的IP加速爬取速度。

单机爬虫 => 分布式爬虫 需要解决的问题

  1. request 队列集中管理：scheduler 以队列形式存储在内存中，而其他服务器无法拿到当前服务器内存中的内容；
  2. 去重集中管理。
    解决方法：将 request 队列和去重 放到第三方组件中，采用 Redis(内存数据库，读取速度更快)。

Redis

Redis 是 key-value 存储系统，数据存在内存中。

Redis 数据类型

字符串 散列/哈希 列表 集合 可排序集合

Redis 字符串常用命令

  1. 添加值 set course_name "math"
  2. 读取值 get course_name
  3. 获取指定区间的数据值 getrange course_name 2 5
  4. 获取字符串长度 strlen course_name
  5. 加/减一 值必须是 int 或 string incr/decr course_name
  6. 字符串追加 append course_name ".com"

Redis 哈希常用命令

  1. 添加值 hset course_name mykey "python"
  2. 读取值 hget course_name mykey
  3. 返回某个变量名的 key value hgetall course_name
  4. 检查是否存在某个key hexists course_name mykey
  5. 删除某个key hdel course_name mykey
  6. 返回key hkeys course_name
  7. 返回值 hvals course_name

Redis 列表常用操作

  1. 向头部添加值 lpush course_name "python"
  2. 向尾部添加值 rpush course_name "python"
  3. 获取指定区间的数据值 lrange course_name 2 5
  4. 删除 若列表为空则等待3秒 b表示阻塞 blpop/brpop course_name 3
  5. 删除 lpop/rpop course_name
  6. 返回列表长度 llen course_name
  7. 取出第2个元素 lindex course_name 2

Redis 集合常用操作

  1. 添加值 sadd course_set "python"
  2. 返回集合中值的总数 scard course_set
  3. 相减 返回差集 sdiff course1 course2
  4. 返回交集 sinter course1 course2
  5. 随机删除并返回 spop course1
  6. 随机获取一个元素 srandmember course_set 1
  7. 获取set中所有元素 smembers course_set

Redis 可排序集合常用命令

  1. 增加值 zadd course_name 0 "math" 5 "eng" 10 "python"
  2. 获取指定区间的元素 zrangebyscore course_name 5 10
  3. 统计指定区间数据量 zcount key 0 100

Scrapy-Redis 编写爬虫需要注意的点

  1. 继承 RedisSpider；
  2. 所有 request 不再由本地 schedule 来完成，而是 Scrapy-Redis 的 schedule；
  3. 需要 push 起始 url。
